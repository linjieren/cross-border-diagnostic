# MOD-004 表单提交后数据追踪

## 用户故事

作为出海企业，我希望确认我的表单转化追踪是否完整部署，以便准确衡量营销效果、追踪用户从表单提交到后续行为的全链路数据。

## Acceptance Criteria

### 检测流程
- [ ] 用户在左侧 iframe 中导航到表单页面，右侧面板自动检测追踪配置
- [ ] 检测项 1：表单提交后是否有 Thank You 页面（独立 URL，非弹窗提示）
- [ ] 检测项 2：Thank You 页面是否部署了转化追踪代码（Google Analytics 4 `gtag('event', 'conversion')` 或 Meta Pixel `fbq('track', 'Lead')` 等）
- [ ] 检测项 3：白皮书/PDF 下载事件是否有追踪代码（`gtag('event', 'file_download')` 等）
- [ ] 检测项 4：页面是否已安装 Google Analytics 4 或 Meta Pixel 基础代码

### 追踪完整性评分
- [ ] **基础追踪**：GA4 或 Meta Pixel 基础代码已安装
- [ ] **表单转化追踪**：Thank You 页面有明确的转化事件代码
- [ ] **下载事件追踪**：PDF/白皮书下载按钮有点击追踪
- [ ] **UTM 参数支持**：表单提交数据是否携带 UTM 来源参数

### 优化建议输出
- [ ] **转化追踪一键部署代码**：
  - GA4 转化事件代码（含粘贴位置：Thank You 页面的 <head> 或 <body> 底部）
  - Meta Pixel Lead 事件代码（含粘贴位置说明）
  - PDF 下载事件追踪代码（含粘贴到下载按钮的说明）
- [ ] **UTM 参数配置指南**：教用户如何在广告链接中添加 UTM 参数，以及在表单后台如何读取
- [ ] **数据验证方法**：如何进入 GA4 实时报告验证转化事件是否正常触发

### 关键指标
- [ ] 追踪部署完整性百分比（如"当前追踪覆盖率 60%，缺少 PDF 下载追踪"）
- [ ] 未部署追踪时给出红色警告和分步安装指南

### 报告集成
- [ ] 本模块的诊断结果自动汇入总报告
- [ ] 报告中包含：追踪覆盖 checklist、部署代码、验证步骤

## 备注
- GA4 检测方式：扫描页面是否包含 `gtag.js` 或 `googletagmanager.com/gtag`
- Meta Pixel 检测：扫描 `connect.facebook.net/en_US/fbevents.js`
- 转化事件检测：扫描页面 JS 中是否包含 `gtag('event', 'conversion')`、`fbq('track', 'Lead')` 等关键词
- 建议同时支持 Google Tag Manager 检测（`googletagmanager.com/gtm.js`）
