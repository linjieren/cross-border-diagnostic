# MOD-002 留资页面检查

## 用户故事

作为出海企业（SME），我希望平台能自动检查我的留资（询盘）页面是否安全、完整且高转化，以便获取真实的海外客户询盘，同时拦截 99% 的机器人和垃圾攻击。

---

## Acceptance Criteria

### 1. 检测流程（自动触发）

用户在左侧 iframe 中导航到留资页面时，右侧面板自动执行以下检测：

#### 1.1 安全防护检测
- [ ] **reCAPTCHA 检测**：扫描页面 HTML/JS 是否包含 `google.com/recaptcha`、`grecaptcha` 相关脚本，区分 v2（Checkbox/Invisible）和 v3
- [ ] **Cloudflare Turnstile 检测**：扫描是否包含 `challenges.cloudflare.com/turnstile` 相关脚本
- [ ] **Honeypot / 隐藏字段检测**：扫描表单中是否存在 `display:none`、`visibility:hidden` 或 `type="hidden"` 的陷阱字段（name 含 honeypot/trip/trap/bot 等关键词）
- [ ] **HTTPS 检测**：确认页面通过 HTTPS 加载，证书有效

#### 1.2 表单结构检测
- [ ] **字段数量和类型分析**：识别所有 `<input>`、`<textarea>`、`<select>` 字段，分类为：姓名、邮箱、电话、公司、国家/地区、职位、预算范围、留言、其他
- [ ] **必填/选填策略分析**：统计 `required` 属性使用情况，识别过度必填（如要求电话+公司+预算同时必填）或过度宽松（全部选填）
- [ ] **提交按钮检测**：确认提交按钮可见、可点击；识别 CTA 文案（如"Submit"、"Send"、"Get Quote"等）
- [ ] **表单提交后反馈**：检测提交后是否跳转 Thank You 页面（独立 URL）或仅显示弹窗/alert

#### 1.3 转化体验检测
- [ ] **页面加载速度**：表单区域首次渲染时间（LCP 中表单元素的渲染耗时）
- [ ] **移动端适配**：表单在 375px 宽度下是否横向滚动、字段是否重叠、按钮是否可点击

### 2. 诊断结果展示（右侧面板）

右侧面板分为三个区块：

#### 2.1 安全评分卡（顶部）
- 安全等级：🟢 安全 / 🟡 需优化 / 🔴 高风险
- 各子项状态图标：reCAPTCHA（是/否/版本）、Turnstile（是/否）、Honeypot（是/否）、HTTPS（是/否）
- 拦截能力预估文字：如"当前配置预计可拦截 99.9% 机器攻击"或"当前无防护，垃圾询盘风险极高"

#### 2.2 表单结构分析（中部）
- 当前字段清单表格：字段名、类型、是否必填、建议
- 字段数量评分：过少（<3 个）/ 适中（3-6 个）/ 过多（>6 个）
- CTA 文案评价：明确（如"Get Free Quote"）/ 模糊（如"Submit"）
- 提交后体验：Thank You 页（优）/ 弹窗（中）/ 无反馈（差）

#### 2.3 优化建议（底部，可折叠）
- 未安装防护时：红色警告 + 推荐方案卡片
- 表单结构问题时：字段调整建议
- 行业模板推荐：按检测到的行业属性匹配最佳模板

### 3. 优化建议输出

#### 3.1 安全防护推荐

**优先级：reCAPTCHA v3（无感知）> Cloudflare Turnstile > reCAPTCHA v2 Checkbox**

- 若未安装任何防护：
  - 主推 reCAPTCHA v3（用户无感知，后台评分 0-1，<0.5 视为机器人）
  - 备选 Cloudflare Turnstile（更轻量，隐私友好）
  - 均附一键部署代码和粘贴位置说明

**一键部署代码示例（reCAPTCHA v3）**：
```html
<!-- 1. 在 <head> 中粘贴以下脚本（将 YOUR_SITE_KEY 替换为实际密钥） -->
<script src="https://www.google.com/recaptcha/api.js?render=YOUR_SITE_KEY"></script>

<!-- 2. 在表单提交逻辑中粘贴以下代码 -->
<script>
grecaptcha.ready(function() {
  grecaptcha.execute('YOUR_SITE_KEY', {action: 'submit'}).then(function(token) {
    // 将 token 附加到表单数据中提交到后端验证
    document.getElementById('recaptchaToken').value = token;
  });
});
</script>
<input type="hidden" id="recaptchaToken" name="recaptchaToken">
```

**一键部署代码示例（Cloudflare Turnstile）**：
```html
<!-- 1. 在 <head> 中粘贴 -->
<script src="https://challenges.cloudflare.com/turnstile/v0/api.js" async defer></script>

<!-- 2. 在表单 <form> 标签内或提交按钮前粘贴 -->
<div class="cf-turnstile" data-sitekey="YOUR_SITE_KEY" data-callback="onTurnstileSuccess"></div>
```

**粘贴位置说明（按建站平台）**：
- **WordPress**：使用 Insert Headers and Footers 插件粘贴到 <head>；或在主题编辑器中编辑 `header.php`
- **Shopify**：Online Store → Themes → Edit code → `theme.liquid` → 粘贴到 `<head>` 标签内
- **Wix**：Settings → Custom Code → 添加到 Head（All Pages）
- **自建 HTML**：直接粘贴到页面 HTML 文件的 `<head></head>` 之间

#### 3.2 行业表单模板

模板至少覆盖以下 4 个行业：

**模板 A：B2B SaaS / 软件服务**
```

1. 名字 [必填] — type="text"
2. 工作邮箱 [必填] — type="email"
```

**模板 B：制造业 / 工业品**

**模板 D：专业服务 / 咨询 / 代理**

```
CTA 文案："Book a Free Consultation"
策略：信息收集充分，预算和时间帮助服务方评估匹配度
```

## 技术边界

### 前端爬虫分析（轻量、实时）
- reCAPTCHA / Turnstile 脚本检测：扫描页面 DOM 中是否存在对应 `<script>` 标签
- Honeypot 检测：扫描表单内的隐藏字段，结合 name 属性关键词匹配
- 字段类型分析：读取表单 DOM 结构，识别 input type、label 文本
- 必填检测：读取 `required` 属性
- 提交按钮检测：查找 `type="submit"` 或 `<button>` 的可见性和文案

### 后端 Puppeteer/Playwright 深度检查（异步、更准确）
- HTTPS 证书有效性验证（前端无法直接获取证书信息）
- 表单提交后行为：提交测试数据后检测是否跳转 Thank You 页（需后端无头浏览器）
- 页面加载性能：测量表单区域 LCP（需性能 API 数据）
- 移动端适配截图：在 375px 视口下截图，供右侧面板展示
- CORS 限制下的跨域检测：所有涉及向目标网站发送请求的检测必须走后端代理

### 注意
- 表单提交测试必须走后端，且使用测试数据（如 `test@example.com`），避免污染客户真实数据
- 若目标网站有 rate limit 或 WAF，检测频率需控制，失败时给出"检测受限，建议手动检查"提示
