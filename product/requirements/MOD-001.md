# MOD-001 全球网站访问加速

## 用户故事

作为出海企业，我希望了解我的网站在目标市场的访问速度，并获得可执行的加速方案，以便海外客户能流畅访问我的网站（延迟控制在 2 秒内）。

## Acceptance Criteria

### 检测流程
- [ ] 用户输入网站 URL + 目标市场后，系统从目标市场地理位置发起访问延迟测试
- [ ] 检测指标：TTFB（首字节时间）、DNS 解析时间、SSL 握手时间、完整页面加载时间
- [ ] 检测完成后右侧面板展示：当前延迟数据 + 与行业基准对比（优秀/<1s、良好/1-2s、差/>2s）
- [ ] 若任何指标 > 2 秒，自动触发优化建议流程

### 优化建议输出
- [ ] **手把手教学视频**：自动抓取 YouTube 上最相关、质量最好的 CDN 配置教程（按观看量+评分排序，取 Top 3）
- [ ] **一键部署代码**：生成 Cloudflare / Akamai CDN 的配置代码片段，含明确的粘贴位置说明（如"粘贴到网站 <head> 标签内"）
- [ ] **账号注册流程指导**：提供 Cloudflare / Akamai 注册步骤的图文/视频引导
- [ ] **加速前后对比预估**：展示部署 CDN 后预计可降低的延迟百分比

### 关键指标验证
- [ ] CDN 优化后海外访问延迟必须降至 2 秒内（系统提供模拟预估，非真实部署后实测）
- [ ] 若网站已部署 CDN，系统检测后给出"已优化"状态及当前 CDN 服务商识别

### 报告集成
- [ ] 本模块的诊断结果自动汇入总报告
- [ ] 报告中包含：延迟数据图表、优化建议清单、教学视频链接、部署代码块

## 备注
- YouTube 视频抓取优先使用 YouTube Data API v3，如 API 受限可降级为页面元数据解析
- 延迟检测可使用自建全球测速节点或第三方服务（如 Pingdom / GTmetrix API）
- CDN 识别可通过响应头中的 `cf-ray`（Cloudflare）、`x-akamai` 等特征判断
