# INF-001 平台基础架构：左右分屏 + iframe 预览 + 报告引擎

## 用户故事

作为平台用户，我希望输入公司网站 URL 和目标市场后，能看到左右分屏的实时诊断界面——左边是网站的 iframe 预览，右边是实时分析面板。我点击网站的任意页面，右侧立即显示该页面的监测过程和结果。最终我可以一键下载专业诊断报告。

## Acceptance Criteria

### 入口页面
- [ ] 首页包含简洁的输入表单：网站 URL（必填，含 URL 校验）+ 目标市场（下拉选择国家/地区）
- [ ] 表单提交后进入诊断工作台
- [ ] 支持常见 URL 格式自动补全（如用户输入 `example.com` 自动补为 `https://example.com`）

### 诊断工作台（左右分屏）
- [ ] 左侧区域：iframe 加载客户网站，占屏幕约 55-60% 宽度
- [ ] 右侧区域：诊断面板，占屏幕约 40-45% 宽度
- [ ] 左侧 iframe 中客户点击任意页面链接时，右侧面板实时响应并加载该页面的对应模块诊断
- [ ] 右侧面板顶部显示当前正在分析的页面 URL 和模块状态指示器（进行中/已完成/未检测）
- [ ] 整体布局在桌面端（≥1280px）稳定显示，移动端提供简化视图或提示"请在桌面端使用"

### 报告引擎
- [ ] 支持一键下载诊断报告，格式：PDF（默认）、Markdown、HTML
- [ ] 报告包含：封面（客户网站、目标市场、诊断时间）、各模块详细结果、可执行方案汇总、外部资源链接汇总
- [ ] PDF 报告排版专业，适合直接发给客户或内部团队
- [ ] 报告中的代码块保持语法高亮和可复制格式

### 状态管理
- [ ] 每个页面的诊断结果缓存在会话中，用户返回已检测页面时直接展示结果（不重复检测）
- [ ] 提供诊断进度总览（4 个模块的完成状态）

### 技术约束
- [ ] iframe 加载第三方网站时需处理 X-Frame-Options 限制（如使用代理加载或友好提示）
- [ ] 页面切换检测需支持 SPA（单页应用）路由变化，不仅是传统链接点击
- [ ] 报告生成应为异步，大报告不阻塞 UI
