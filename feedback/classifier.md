# 反馈分类与优先级规则

## 分类规则

| 类别 | 判断标准 | 示例 |
|------|----------|------|
| bug | 功能异常、报错、崩溃、数据错误 | "点了保存没反应"、"页面白屏" |
| ui_ux | 界面显示问题、交互不顺畅、文案错误 | "按钮颜色看不清"、"操作步骤太多" |
| feature_request | 用户建议新增功能或能力 | "希望能批量删除"、"加个导出功能" |
| performance | 卡顿、加载慢、响应延迟 | "打开要等 5 秒"、"列表滚动很卡" |
| other | 无法归入以上类别的反馈 | 一般咨询、好评、吐槽 |

## 优先级规则

| 优先级 | 判断标准 | 响应要求 |
|--------|----------|----------|
| p0_critical | 核心功能不可用、数据丢失、安全漏洞 | 立即 notify_parent 升级 |
| p1_high | 严重影响体验，大部分用户会遇到 | 下一轮评估后推动 |
| p2_medium | 部分用户体验下降，有 workaround | 排入迭代队列 |
| p3_low | 小问题、锦上添花的建议 | 累积到一定数量后批量评估 |

## 去重规则

1. 计算反馈文本的标准化指纹（去标点、小写、去多余空格）
2. 与已有反馈指纹库比对
3. 相似度 >= 80% 判定为重复
4. 重复反馈标记 duplicate_of 并记录频次，不重复创建需求
